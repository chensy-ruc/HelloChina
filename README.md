# WanderChina

一个面向在华留学生的多语言中国旅游导览网站。

项目以“地图优先”的方式组织内容：用户先在中国地图上选择旅游城市，再进入该城市的景点地图、景点列表、图片画廊和推荐路线。整个页面围绕“快速理解一座城市、快速选择想去的景点、快速分享给别人”来设计，含有多语言版本，解决在华留学生“No Entrance to Trip in China”的痛点。

## 项目定位

`WanderChina` 的目标用户是在中国学习、生活、旅行的国际学生。相比普通旅游网站，这个项目更强调：

- 用地图帮助用户快速建立城市空间感
- 用多语言界面降低信息门槛
- 用本地景点图片增强展示效果
- 用推荐路线帮助用户组织真实出行顺序

## 项目亮点

- 中国地图总览
  用户可以在全国地图上看到 5 个重点旅游城市，并通过点击城市直接进入对应导览。

- 城市地图联动
  切换城市后，城市地图、景点快捷入口、景点详情、推荐路线和景点列表会同步更新。

- 景点位置标注
  每个城市的景点都会在城市地图中显示位置，并带有编号与名称提示，便于用户快速定位。

- 多语言支持
  当前支持 `EN / 中文 / JA / KO / ES / FR / AR` 七种语言，不仅切换按钮文案，也切换城市、景点、路线和说明文本。

- 本地图片接入
  项目完整使用 `景点图片` 文件夹中的本地资源，适合离线整理和后续继续补充素材。

- 展示型视觉设计
  页面采用明快浅色主题，加入轻量悬浮、渐变光感和卡片动画，适合展示而不过分花哨。

## 当前页面结构

### 1. 首屏 Hero

- 介绍项目主题
- 展示城市数量、景点数量、语言数量
- 提供快速进入导览和随机城市入口

### 2. 中国地图总览区

- 展示 5 个重点旅游城市
- 点击城市后会直接切换到该城市导览
- 左侧同步显示当前城市简介和亮点标签

### 3. 城市工作区

- 展示当前城市地图
- 标出城市内景点位置
- 支持点击地图景点标记切换当前景点
- 支持根据当前景点生成动态路线

### 4. 景点焦点区

- 展示当前景点名称和介绍
- 展示景点主题、推荐时间等信息
- 展示该景点对应的本地图片画廊

### 5. 推荐路线区

- 每个城市包含多条静态主题路线
- 页面还会根据用户当前点击的景点自动生成一条动态路线
- 路线内容会与城市地图联动显示

### 6. 景点卡片列表区

- 展示当前城市全部景点
- 支持卡片点击切换
- 与地图、图片、路线保持同步

## 交互逻辑

整个网站是联动式设计，不是割裂的多个页面。

用户操作任意一个入口时，页面会整体更新：

- 点击中国地图城市
  切换到该城市地图，并同步更新景点列表、路线和焦点景点

- 点击城市地图景点
  切换当前景点，并更新景点介绍、图片画廊和动态路线

- 点击景点快捷入口或景点卡片
  同样会更新地图高亮、焦点区和推荐路线

## 技术实现

项目是纯静态前端网站，不依赖后端服务。

### 主要技术

- `HTML`
- `CSS`
- `JavaScript`
- `Leaflet`
- `OpenStreetMap`

### 实现方式

- `index.html`
  页面结构与主要内容容器

- `styles.css`
  视觉风格、响应式布局、动效、地图与卡片样式

- `app.js`
  城市数据、景点数据、地图逻辑、语言切换、交互联动、路线生成

- `景点图片/`
  各城市景点的本地图片资源

## 项目内容来源与组织方式

项目中的景点展示并不是随机写死，而是围绕本地图片素材组织：

- 每个城市有独立的景点图片目录
- 每个景点可对应多张本地图片
- 前端通过数据脚本将景点名称、坐标、介绍和图片统一组织
- 页面展示时优先使用本地图片资源

这种结构适合后续扩展。如果以后你想继续增加城市或增加景点，只需要：

1. 补充图片目录
2. 在 `app.js` 中补充城市和景点数据
3. 页面会继续沿用现有联动结构

## 本地运行

在项目根目录启动一个静态服务器：

```bash
python3 -m http.server 8000
```

然后在浏览器中打开：

```text
http://localhost:8000
```

## 项目总结

`WanderChina` 不是单纯的景点罗列页，而是一个围绕“城市选择 -> 地图定位 -> 景点浏览 -> 路线组织”建立起来的多语言旅游导览网站。它兼顾了展示效果、交互完整性和后续扩展空间，比较适合用作一个面向国际用户的中国城市旅游可视化项目。
